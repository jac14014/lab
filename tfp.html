// Protocol State Machine

Five states.
Zero discretion.

TFP operates as a deterministic state machine — proximity band determines instrument set, authority level, and response obligation. No governance gap between signal and action.

● MONITOR
Baseline Operation
Continuous monitoring, institutional readiness maintenance, data collection. No instrument activation required.
P(Θ) < α₁
● ALERT
Enhanced Monitoring
Stakeholder notification, preparatory measures activated, enhanced data collection cycle, preliminary instrument readiness.
P(Θ) ≥ α₁
● PRECAUTION
Precautionary Instruments
Legal precautionary instruments activated — resource mobilisation, access restrictions, mandatory reporting obligations, elevated authority council convened.
P(Θ) ≥ α₂
● INTERVENTION
Full Instrument Deployment
Complete intervention instrument set deployed, inter-institutional coordination mandated, designated intervention authority assumes governance primacy.
P(Θ) ≥ α₃
● EMERGENCY
Emergency Protocols
Maximum instrument deployment, emergency governance authority, all cross-scale coordination obligations activated, threshold crossing imminent.
τ(P) < τ_critical
// Protocol Components

What TFP
specifies.

The TFP is not a single document but an architecture — six interdependent components that together define how threshold proximity becomes governance action.

Component 01
Trigger Catalogue

Structured catalogue mapping threshold proximity conditions to specific governance triggers — each trigger defines the institutional actor, authority scope, and instrument type.

Component 02
Legal Instrument Specifications

Detailed specifications for each legal instrument — legal basis, scope, duration, deactivation conditions, and review requirements for each proximity state.

Component 03
Institutional Authority Matrix

Mapping of actors, their authorities at each TFP state, decision procedures, and coordination obligations across governance scales.

Component 04
TFP–TDR Interface

Technical specification for the data interface between TDR proximity models and TFP activation logic — ensuring instruments receive accurate, timely proximity signals.

Component 05
Calibration Parameters

Domain-specific values for activation thresholds α₁, α₂, α₃ and minimum time-to-threshold τ_min — derived from TDR outputs and validated against field data.

Component 06
Audit & Traceability

Institutional logging requirements, decision audit trails, and accountability mechanisms for every activation and deactivation event in the TFP lifecycle.
